Overview

Each of these only relies on the previos ones, so you can pick the level of functionality you want:

SVGKit.js -- A low-level JavaScript SVG library.  Create a new SVG at runtime.  Load SVG data from a URL. Conveniently handle quirks of three ways SVG can be included in browsers: object, embed, and inline. Handle differnet browsers and different SVG engines.  Has convenience functions for DOM manipulation ala MochiKit. It manages the objects: HTMLElement, SVGDocument, SVGElement.  Basic intaractive event handling like scroll, zoom, object select and manipulate (maybe this should go in SVGEvent.js)

SVGCanvas.js -- Impliments some of the Canvas API along with some SVG-specific additions. Allows you to create an SVG more prodecurally rather than building it up with DOM manipulation (one of the disadvangages sighted by Vladimir VukiÄ‡eviÄ‡) There are parts of the Canvas API that do things like "take what you've drawn so far, treat it like a bitmap, and run the following filter on it."  Some of these are impossible in SVG, but will get better with SVG1.2

SVGPlot.js -- Eventually a full plotting suite to rival Excel, Matlab, etc. It generates clean SVG in the browser that can be saved, sent back to server for conversion to PDF, PS, PNG, etc. Fully interactive -- zooming, selecting data, etc. Live updates via AJAX for real-time data like stocks or web stats. Allows multiple plots arranged on an SVG, multiple related or unrelated axes per plot, and control of styles. Origionaly I was going to write it in pure Canvas code so you could have either underneath (like PlotKit), but things like text made it impossible.

History

I wanted a way to do interactive web stuff for my brainflux.org page of physics and math education. I looked into SVG and played around with Batik a bit. Then one of my research groups implimented a full web app using Python, TurboGears, and MochiKit. We wanted to interactively pull data out of a web page and plot it. Also we wanted a Google Maps type interface for stars. I started really working on this in January 2006.

Things I want to do with it

  • More interactivity --  drag/zoom controls,  selecting & manipulation of objects...
  • Clean up the SVG that SVGCanvas and SVGPlot generate and make them faster.  These are sometimes at odds.
  • Interactive math/physics educational stuff like Compass & Ruler
  • Online interface to create and edit plots -- position ledgend, click to edit title, select colors with GUI, etc.

SVG Complaints/Outlook:

Fonts are a huge problem. The SVG spec is written so that you can embed a font and use it so you get precise alignment and pixel-perfect anti-aliased rendering. When you're rendering axis labels, I query the exact dimensions of the resulting text for optimal placement.  None of this works (very well) in Firefox yet.
